# 모션그래픽 웹사이트 — 프로젝트 계획

> 움직이는 그래픽이 핵심인 웹사이트를 만들고, Vercel로 URL을 공개하는 계획서  
> 작성일: 2026-06-27

---

## 이 프로젝트에서 만들 것

| 항목 | 내용 |
|------|------|
| **목표** | 스크롤·호버·등장 등 모션이 살아 있는 웹사이트 |
| **공개 URL** | `https://[프로젝트이름].vercel.app` (나중에 내 도메인 연결 가능) |
| **현재 상태** | 빈 프로젝트 — 처음부터 시작 |

---

## 전체 흐름 (한눈에 보기)

```
① 기획 & 디자인 방향 정하기
        ↓
② 사이트 뼈대 & 화면 구성
        ↓
③ 모션 하나씩 넣기
        ↓
④ 콘텐츠(이미지·영상·텍스트) 채우기
        ↓
⑤ 마무리 점검
        ↓
⑥ Vercel에 올려 URL 받기
```

---

## 1단계 — 기획 & 디자인 방향 (1~2일)

### 결정할 것

- [ ] **사이트 유형** — 아래 중 하나 선택
  - **포트폴리오형** — 작업물을 보여주는 쇼케이스
  - **랜딩페이지형** — 하나의 메시지·브랜드를 강하게 전달
  - **실험·아트형** — 모션 자체가 주인공인 작품 페이지

- [ ] **톤 & 무드** — 예: 미니멀 / 다크 / 네온 / 에디토리얼 / 플레이풀

- [ ] **색 & 폰트**
  - 메인 컬러 1~2개, 배경색, 텍스트색
  - 제목용 폰트, 본문용 폰트 (Google Fonts 무료 사용 가능)

- [ ] **레퍼런스 3~5개** — 마음에 드는 사이트 URL 또는 스크린샷 모음  
  - 참고: [Awwwards](https://www.awwards.com), [Codrops](https://tympanus.net/codrops/)

### 이 단계가 끝나면

- Figma(또는 종이)에 **첫 화면(Hero) 스케치** 1장
- **섹션 목록** — Hero → Works → About → Contact 등 순서 확정

---

## 2단계 — 사이트 뼈대 만들기 (2~3일)

> 코드는 AI·개발 도구가 세팅. 디자이너는 **화면 구조와 간격**을 확인하면 됩니다.

### 사용할 기술 (참고용 — 직접 만질 필요 없음)

| 역할 | 도구 | 왜 쓰는지 |
|------|------|-----------|
| 사이트 기본 틀 | Next.js | Vercel과 잘 맞고, 빠른 페이지 로딩 |
| 화면 꾸미기 | Tailwind CSS | 레이아웃·간격·반응형을 빠르게 맞춤 |
| 정밀한 모션 | GSAP | 스크롤 연출, 타임라인, 복잡한 시퀀스 |
| UI 모션 | Framer Motion | 버튼·카드·페이지 전환 등 |
| 부드러운 스크롤 | Lenis | 스크롤 기반 모션과 자연스럽게 연결 |

### 화면 구성 (MVP — 최소 버전)

**한 페이지로도 충분합니다.** 아래 섹션을 위에서 아래로 쌓는 형태:

```
┌─────────────────────────┐
│  Hero (첫 인상 + 메인 모션)  │
├─────────────────────────┤
│  Works (작업물 갤러리)       │
├─────────────────────────┤
│  About (선택)             │
├─────────────────────────┤
│  Contact (연락처)          │
└─────────────────────────┘
```

페이지를 나누고 싶다면: `/works`, `/about` 등 추가 가능 (나중에 확장)

### 이 단계가 끝나면

- [ ] 데스크톱·모바일에서 **레이아웃이 깨지지 않는지** 확인
- [ ] 아직 모션 없이도 **타이포·색·간격**이 의도대로인지 확인

## 3단계 — 모션 넣기 (5~7일)

> 모션은 **한 섹션씩** 추가하고, 매번 브라우저에서 결과를 보며 조정합니다.

### 섹션별 모션 아이디어

| 섹션 | 사용자가 느끼는 것 | 구현 방향 |
|------|-------------------|-----------|
| **Hero** | 페이지 열자마자 타이틀이 부드럽게 등장, 배경이 살아 있음 | 텍스트 순차 등장, 그라데이션·파티클 |
| **스크롤 Reveal** | 아래로 내리면 콘텐츠가 fade / slide 로 나타남 | 스크롤 위치에 맞춰 등장 |
| **Works 갤러리** | 카드에 마우스를 올리면 확대·이미지가 살짝 움직임 | hover + 패럴랙스 |
| **마이크로 인터랙션** | 버튼·링크에 반응하는 작은 움직임 | hover, 커스텀 커서(선택) |
| **페이지 전환** | 다른 페이지로 갈 때 끊김 없이 넘어감 | 페이드·슬라이드 |
| **Footer** | 텍스트가 흐르거나 선이 그려지는 장식 | marquee, 라인 드로잉 |

### 모션 작업 순서 (권장)

1. Hero 진입 애니메이션  
2. Lenis 부드러운 스크롤  
3. 섹션별 스크롤 reveal  
4. Works 호버·갤러리 모션  
5. 버튼·링크 등 작은 디테일  
6. (선택) 페이지 전환, Footer 장식  

### 디자인 체크 포인트

- [ ] **모바일**에서도 모션이 과하지 않은지 (너무 많으면 답답함)
- [ ] **속도· easing** — 너무 빠르거나 느리지 않은지
- [ ] **움직임 줄이기 설정** — OS에서 “애니메이션 줄이기” 켜진 사용자도 불편하지 않게 (`prefers-reduced-motion` 대응)

### (선택) 외부 모션 에셋

| 도구 | 언제 쓰나 |
|------|-----------|
| **Lottie** | After Effects로 만든 애니메이션을 웹에 넣을 때 |
| **Rive** | 인터랙티브 벡터 애니메이션 |
| **Three.js** | 3D 모션·입체 그래픽 |

## 4단계 — 콘텐츠 채우기 (2~3일)

### 준비할 자료

- [ ] **이미지** — WebP 권장, 큰 파일은 압축 (작업물 썸네일, OG용 대표 이미지)
- [ ] **영상** — 짧은 루프는 WebM/MP4, 자동재생은 muted + loop
- [ ] **텍스트** — 제목, 설명, 연락처(이메일, SNS 링크)
- [ ] **파비콘** — 브라우저 탭 아이콘
- [ ] **OG 이미지** — 링크 공유 시 미리보기 (1200×630px 권장)

### Works(작업물) 정리

각 작업마다 정리할 항목 예시:

```
- 제목
- 짧은 설명
- 썸네일 이미지 (또는 영상)
- (선택) 상세 페이지 링크 / 외부 URL
- (선택) 연도, 카테고리
```

## 5단계 — 마무리 점검 (1~2일)

### 보기·느끼기 관점

- [ ] **Chrome, Safari, Firefox**에서 동일하게 보이는지
- [ ] **iPhone / Android** 화면 크기에서 레이아웃·모션 확인
- [ ] **로딩** — 첫 화면이 너무 오래 비어 있지 않은지
- [ ] **링크 공유** — 카카오·슬랙 등에 붙였을 때 미리보기(OG)가 예쁜지

- Lighthouse Performance **80점 이상** (가능하면)
- 이미지·영상 lazy load — 스크롤 전까지 무거운 파일 안 불러오기

## 6단계 — Vercel 배포 (반나절)

> 코드를 GitHub에 올리면, Vercel이 자동으로 빌드하고 **공개 URL**을 만들어 줍니다.

### 필요한 계정 (모두 무료로 시작 가능)

| 서비스 | 하는 일 |
|--------|---------|
| **GitHub** | 코드 저장소 — Vercel과 연결 |
| **Vercel** | 웹 호스팅 — URL 발급 & 자동 배포 |

### 배포 순서

1. [github.com](https://github.com) — 저장소(repository) 만들기  
2. 프로젝트 코드 push  
3. [vercel.com](https://vercel.com) 가입 → **GitHub 계정 연동**  
4. Vercel Dashboard → **Add New Project**  
5. 방금 만든 GitHub 저장소 선택 → **Import**  
6. Framework: **Next.js** (자동 감지됨) → **Deploy**  
7. 몇 분 후 `https://○○○.vercel.app` URL 생성 완료  

### 배포 후 자동으로 되는 것

| 상황 | 결과 |
|------|------|
| `main` 브랜치에 코드 push | **Production** — 실제 공개 URL 업데이트 |
| PR(수정 요청) 만들기 | **Preview URL** — 수정본만 미리 볼 수 있는 임시 링크 |

### (선택) 내 도메인 연결

1. Vercel → Project → **Settings → Domains**  
2. `www.mysite.com` 입력  
3. 도메인 구매처(DNS)에서 안내하는 레코드 추가  
4. SSL(자물쇠 🔒) 자동 적용
